Critical — Fix These First

  1. Hardcoded passwords in the code
  In server/db.ts, two accounts with plaintext passwords are baked into
  the source code (winefan1992 and AppleReview2025!). Anyone who sees your
   code can log in. Move these to a separate seed script that isn't in
  production.

  2. Google Sign-In doesn't actually verify identity
  In server/auth.ts, when someone signs in with Google, the app accepts
  whatever email and Google ID they send without verifying it with
  Google's servers. This means anyone could claim to be any user. Use
  Google's token verification API to validate the id_token before trusting
   it.

  3. Schema mismatch causing silent data loss
  The client-side schema (shared/schema.ts) has fields called store and
  bin on bottles, but the actual database only has a location field. Any
  code reading or writing store/bin silently fails. Either add these
  columns to the database or remove them from the schema.

  ---
  High — Should Fix Soon

  4. CSV import leaves data in a broken state on failure
  In server/routes.ts, if a CSV import fails halfway through, there's no
  rollback — you get a partial import with no clear way to recover. Wrap
  the import in a proper database transaction that rolls back everything
  if any row fails.

  5. Deleting an account permanently destroys all wine history
  There's no soft delete or backup when a user deletes their account. All
  wines, bottles, and consumption logs are gone forever. Add a
  confirmation step and consider keeping data for 30 days before hard
  deletion.

  6. No rate limiting on login
  Anyone can attempt unlimited login attempts. Add rate limiting to the
  login, register, and password endpoints to prevent brute force attacks.

  ---
  Medium — Worth Addressing

  7. Season calculations are off by one month
  In server/ai-tools.ts, the getSeason function uses JavaScript's
  getMonth() which returns 0–11 (January = 0), but the code treats it as
  1–12. Seasonal wine recommendations will be wrong by one month.

  8. Wine photo scanning isn't wired up
  There's a working /api/analyze-wine-image endpoint on the server, but
  the camera UI in the app never actually calls it. The feature appears to
   exist but doesn't function end-to-end.

  9. Slow query on the wine list
  The main wine list query runs a separate database lookup for every
  single wine to get its storage location. With a large cellar this will
  get slow. Rewrite it to get the location in the main query.

  10. Chat could hang indefinitely
  The AI sommelier chat has no timeout. A complex request could block the
  connection forever. Add a timeout (e.g. 60 seconds) that gracefully
  closes the stream.

  ---
  Minor — Cleanup

  11. Drizzle ORM is installed but never used
  drizzle-orm and drizzle-kit are in package.json but the app uses raw
  SQLite queries everywhere. Remove these unused dependencies.

  12. server/storage.ts is an empty file
  It only contains export { db } and nothing imports it. Safe to delete.

  13. Database file location isn't configurable
  The SQLite file path is hardcoded. Make it an environment variable so it
   can be set to a persistent path in production.
